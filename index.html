Documentation de l'API REST - Democratia

Structure des requêtes

URL : rest.php?request=NOM_ACTION parameters=["VAL1", "VAL2"]

  • request : Nom de l'action à faire dans l'API.
  • parameters : Tableau encodé contenant les valeurs dans l'ordre de la procédure. ou de l'apparition des points d'intérogation

Exemple : rest.php?request=infoInternaute parameters=["3"]

Pour des questions de lisibilité, il y a pas le & mais il est évidemment à inclurer dans la requete, en collant tous les caractères

🔐 Requêtes SQL Personnalisées

Règle de sécurité : Le mot-clé SQL doit correspondre à la méthode HTTP. Toutes les requêtes personnalisées doivent obligatoirement utiliser des paramètres préparés (?).
Verbe HTTPMot-clé SQL requisExemple de request
GETSELECTSELECT * FROM internaute WHERE id=?
POSTINSERTINSERT INTO groupe (nom) VALUES (?)
PATCHUPDATEUPDATE internaute SET mail=? WHERE id=?
DELETEDELETEDELETE FROM proposition WHERE id=?

📋 Auto-documentation

Pour obtenir la liste de toutes les méthodes intégrées au serveur :

GET rest.php?request=getMethode

Aucun paramètre supplémentaire n'est requis pour cette commande.
